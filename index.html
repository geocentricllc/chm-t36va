Set a non-zero value to slow this specific part down — useful for:
• Tall or top-heavy parts (electrolytics, modules) that vibrate or shift if moved fast
• Fragile parts (large QFNs, fine-pitch BGAs) where placement accuracy matters more than throughput

Quick start

        
    
          
  1. Drop your Fusion 360 .mnt (or CSV) onto the dropzone.
    2. 
          
  2. Continue to the Assign Feeders table — assign a station ID to each part group, set nozzle, tape pitch, and height.
    3. 
          
  3. Continue to Board / Calib — set X/Y offset if your file's origin isn't the PCB's lower-left corner.
    4. 
          
  4. Generate DPV, review the preview, click Download.
    5. 
          
  5. Load the .dpv on the CHM-T36VA via USB stick. Calibrate fiducials on the machine before running.
    6. 
        

        
That's the whole flow for a typical board. The rest of this help expands on each piece.


        
Supported file formats


        
Fusion 360 .mnt and .mnb

        
These are Fusion's native pick-and-place outputs. Generate them via Manufacturing Workspace → Outputs → Pick and Place. Fusion writes one file per side: .mnt = top, .mnb = bottom. The column layout is fixed:

        
RefDes  LocationX  LocationY  Rotation  Value  Package

        
Whitespace-delimited, no header line, comments starting with # / ; / * are stripped. The converter recognizes the extension and skips column-detection automatically.


        
CSV / TXT pick-and-place

        
Generic comma/tab/semicolon-delimited files from KiCad, EAGLE, Altium, or Fusion's CAM Processor work too. Column detection is based on header-row name matching:

        

          
FieldRecognized header namesRequired?

          
            
ReferenceDesignator, RefDes, Reference, Name, Part, ComponentYes

            
ValueValue, Comment, Val, DescriptionNo

            
PackagePackage, Footprint, Pattern, PkgNo

            
X coordMid X, X, Center X, Loc X, Pos X, Ref XYes

            
Y coordMid Y, Y, Center Y, Loc Y, Pos Y, Ref YYes

            
RotationRotation, Rot, Angle, Orientation, ThetaYes

            
SideSide, Layer, TB, MirrorNo

            
DNPDNP, DNI, Do Not Place/Populate/Install, PopulateNo

          
        

        
The first data row is detected automatically — preamble lines like [COMMENT], Centroid Data for…, and [PLACEMENT] are skipped before the header is identified.


        
Coordinate units

        
Auto-detected from the magnitude of the X/Y values:

        
    
          
  • Max coordinate < 13 → inches
    • 
          
  • Max coordinate ≤ 300 → millimeters
    • 
          
  • Max coordinate > 300 → mils (1/1000 inch)
    • 
        

        
The override dropdown on Step 1 lets you change this if the heuristic gets it wrong (rare, but possible for very small boards measured in mils).


        
Workflow walkthrough


        
Step 1 — Load file

        
Drop or click to browse. After parsing, you'll see file stats and an "Auto-detected — override only if wrong" panel with three dropdowns and a one-line summary of which columns mapped to what. Most boards just need Continue.

        
When to touch the overrides:

        
    
          
  • Heuristic picked the wrong units (small mil-units board misread as mm)
    • 
          
  • File has both top and bottom parts and you want both — change Side filter to "All rows"
    • 
          
  • You want to include parts marked DNP — set DNP filter to "Include them"
    • 
        


        
Step 2 — Assign feeders

        
The table shows one row per placement on your board. Identical part types (same Value + Package) share feeder/nozzle/tape settings — editing those on one row updates every row of that part type. Editable per row:

        
    
          
  • X / Y — placement coordinates in mm (per-placement)
    • 
          
  • Rotation — placement angle in degrees, CCW positive (per-placement)
    • 
          
  • Feeder — station ID where this reel is loaded (1–99) — shared across same-type parts
    • 
          
  • Nozzle — N1 (left) for small parts, N2 (right) for larger ones — shared
    • 
          
  • Tape pitch — physical reel spacing in mm; see tape pitch reference — shared
    • 
          
  • Height — physical thickness of the part (mm) — shared
    • 
          
  • Speed — 0 means use machine default; raise above 0 only if the part needs slower handling — shared
    • 
          
  • Vision — on for ICs and fine-pitch; can turn off for very small passives — shared
    • 
          
  • Skip — shared; flags the entire part type as "don't place" in the DPV
    • 
        

        
Click any column header (RefDes, Value, Package, X, Y, Rotation) to sort. Click again to reverse, third time to return to insertion order. RefDes uses natural sort (R1 < R2 < R10).

        
A subtle background tint clusters consecutive rows of the same part type, making it easy to scan groupings even when sorted by RefDes or coordinate.

        
The table is paged at 20 rows per page. Use Prev / Next or type a page number into the jump box at the bottom of the table to navigate. Sort applies to the full dataset before paging, so sorting jumps you to page 1 with the right rows on top.

        
To exclude a part type from the run, tick its Skip checkbox — that row's placement still appears in the table (dimmed) and the DPV will mark it as skipped so the machine moves over it. Editing happens in-place; there is no separate save step.


        
Step 3 — Board origin & calibration

        
If your CSV's coordinates aren't relative to the PCB's lower-left corner, set X/Y offsets here. See board origin for details. Fiducial points are optional — the operator typically calibrates them on the machine.


        
Step 4 — Review & export

        
The DPV preview shows exactly what will be written. Filename is editable. Download saves a plain-text .dpv file you can copy to a USB stick and load on the CHM-T36VA.

Nozzle selection

        
The CHM-T36VA has two nozzles mounted side by side, named N1 (left) and N2 (right). Each can be a different physical nozzle (CN040, CN065, CN140, etc.) — you switch nozzles on the machine when changing what's mounted.

        
    
          
  • N1 (left) — typically a smaller nozzle for 0201, 0402, 0603, SOT-23. Default for passives.
    • 
          
  • N2 (right) — typically a larger nozzle for SOIC, QFP, QFN, BGA, electrolytics. Default for ICs.
    • 
        

        
The converter auto-picks based on package name. If you only have nozzles loaded for one head and want everything on N1, edit each row or sort by Package and bulk-edit (one row at a time — bulk editing isn't a feature yet).

Board origin & rotation

        
The CHM-T36VA places parts using coordinates relative to the PCB's lower-left corner. After fiducial calibration, the machine knows where that corner is in machine coordinates and translates accordingly.

        
Fusion 360 typically exports with the same convention, so X/Y offsets of 0,0 work in most cases.

        
You'll need to set offsets when:

        
    
          
  • Your design uses board-center origin instead of corner origin
    • 
          
  • Negative X or Y values appear in the placements (the machine's origin must be set so all placements are positive)
    • 
          
  • You're panelizing one-up boards into a larger panel and want to shift the placements into the panel's coordinate system
    • 
        

        
Quick test: after Step 2 the table displays placements in the order they'll be made. If the smallest X/Y values are near zero — good. If they're at, say, X = −18 to +18 — the file uses a centered origin and you need an offset of about +18 mm.

        
Global rotation offset (Step 3) rotates every placement by the same angle. Useful if you want to run the same DPV on a board mounted in a different orientation; rare otherwise. For individual reel-vs-footprint orientation mismatches, edit the affected placements' Rotation column directly on Step 2.


        
Fiducials & calibration

        
Fiducials are reference marks on the PCB that the machine's camera uses to figure out exactly where your board is sitting on the bed. The CHM-T36VA expects three:

        
    
          
  • UL — upper-left fiducial
    • 
          
  • LR — lower-right fiducial
    • 
          
  • LL — lower-left fiducial
    • 
        

        
You can leave the fiducial X/Y values at 0 in the converter. The standard practice is to calibrate them at the machine itself: load the DPV, jog the camera to each physical fiducial mark, and the machine records the actual position into the file's CalibPoint section. This makes the converter's fiducial fields effectively optional — they're a starting hint at best.

        
If you do enter fiducial coordinates, use the same coordinate system as your placements (same units, same origin) so the machine has a consistent frame of reference.


        
DPV file format

        
The .dpv file is plain text, CRLF line endings, structured as labeled sections. The CHM-T36VA reads it from a USB stick. Generated files have these sections in order:

        
    
          
  1. separated + header (FILE, PCBFILE, DATE, TIME, PANELYPE)
    2. 
          
  2. Station rows — one per non-skipped part group
    3. 
          
  3. Panel_Array — single board (1×1) by default
    4. 
          
  4. EComponent rows — one per placement (RefDes-level)
    5. 
          
  5. PcbCalib — placeholder (set on machine)
    6. 
          
  6. CalibPoint rows — three fiducial entries
    7. 
          
  7. CalibFator — placeholder (computed by machine)
    8. 
        

        
Coordinates are written with 2 decimal places (the machine truncates further digits anyway). Angles are normalized to (−180°, 180°].

        
Spec quirk: the machine requires at least 2 EComponent rows or it can't compute fiducial calibration. If your job has only one placement, the converter pads with a phantom skipped row to satisfy this requirement.

Saving feeder configs

        
The ↑ Save config button on Step 2 exports a JSON file with all your feeder assignments — keyed by Value+Package. The ↓ Load config button applies a saved config to the current board.


        
How matching works

        
Each entry in the saved JSON is identified by its Value and Package. When you load a config onto a different board:

        
    
          
  • If the new board has a part group with the same Value+Package (case-insensitive), the saved settings are applied to that group.
    • 
          
  • If the new board has a part group not in the config, it keeps its auto-guessed defaults.
    • 
          
  • If the config has entries for parts not on the new board, they're listed as "unused" in the summary.
    • 
        

        
Settings saved per part type: feeder, nozzle, tape pitch, height, speed, vision flag, skip flag. Not saved: per-placement coordinates and rotations (those are board-specific).


        
Workflow suggestion

        
Tune feeder assignments carefully on your first board. Save the config. On every subsequent board, load the config first and tweak only the new parts. Per-placement rotation tweaks (for reel-vs-footprint orientation fixes) can be applied directly on Step 2 — but those are placement-specific, so if every R1/R2/R3 of a reel needs the same +90° fix, set it once in your CAD library footprint instead so it comes through correctly from the .mnt.


        
Troubleshooting


        
"Couldn't find required column(s)"

        
Auto-detection couldn't identify a required column (RefDes, X, Y, or Rotation) from the header row. Most common causes:

        
    
          
  • File has no header row — open in a text editor and check.
    • 
          
  • Headers use unusual names — supported names are listed in file formats.
    • 
          
  • File has a long preamble that's confusing the header detector. The detector scans the first 15 non-comment lines and picks the most header-like row, but unusual structures can trip it.
    • 
        


        
"After filtering, 0 placements remain"

        
The diagnostic alert breaks down where every row went. Most common causes:

        
    
          
  • Side filter mismatch — your file uses Side values like F.Cu or unusual numbers. Set Side filter to "All rows" or set the Side column to "none".
    • 
          
  • All rows have non-numeric coordinates — file might have units appended ("12.5mm"); the parser strips common unit suffixes but not all.
    • 
        


        
Rotations are off by 90° or 180° on certain parts

        
Almost always a reel-vs-footprint orientation mismatch. Two ways to fix:

        
    
          
  • Best (permanent): fix the footprint in your CAD library so future boards come through correctly.
    • 
          
  • Quick (per-board): sort the table by Value or Package, edit the Rotation column on each affected row to add 90 / 180 / -90 as needed. Tedious for many placements, fine for a few.
    • 
        


        
The machine reports placements off the board

        
Usually a coordinate origin issue. Check Step 3's X/Y offset. If your Fusion file uses board-center origin, the offset should be about (half board width, half board height).


        
"Duplicate feeder IDs assigned"

        
You've assigned the same feeder number to two different part groups. The DPV will still be generated but the machine will physically only have one reel at that station — the second part group will pick the wrong reel. Re-assign one of them to an unused feeder ID.
